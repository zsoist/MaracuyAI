# Project Status

## Snapshot

MaracuyAI is currently a working local prototype and applied ML portfolio artifact.

## What Works Today

- FastAPI backend with upload, analysis, auth, and persistence
- local browser dashboard served by the backend
- Expo mobile client with recording and history flows
- audio normalization and storage
- binary inference path when a trained local `.keras` model is available
- fallback repo inference path for development continuity

## What The Repo Proves

- end-to-end engineering across model, API, storage, and client layers
- serious experimentation on a narrow audio-classification problem
- collaborative technical work around a real-world use case

## What The Repo Does Not Yet Prove

- benchmarked model quality
- production readiness
- veterinary or medical diagnostic validity
- generalized multi-bird or species-level performance

## Current Maturity

- `Prototype`: yes
- `Local demo ready`: yes
- `Model experimentation present`: yes
- `Evaluation rigor hardened`: not yet
- `Production validated`: no

## Most Important Next Steps

1. freeze dataset splits and publish evaluation artifacts
2. simplify all user-facing surfaces around binary output
3. add automated tests and model regression checks
4. add demo screenshots / GIFs for easier external review
