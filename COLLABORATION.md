# Collaboration Framing

## Why This File Exists

MaracuyAI is strongest when described honestly as collaborative technical work around a real-world problem. This file exists to keep public framing disciplined and credible.

## Honest Project Framing

Use language like:

> MaracuyAI is a collaborative applied ML prototype developed around a personalized bird-audio classification use case.

Avoid language like:

- "fully production-validated AI platform"
- "medical diagnostic system"
- "solo breakthrough model" unless authorship is explicitly documented elsewhere

## Collaboration Shape

The repository supports a collaborative story in which:

- the use case came from a real caretaker / bird context
- labeling and interpretation required human judgment outside pure software work
- engineering work translated that context into a usable prototype

That is a strength, not a weakness. It shows the ability to work with real users, ambiguous data, and evolving requirements.

## What The Repo Most Credibly Demonstrates

- problem reframing from broad scope to narrow binary classification
- applied ML workflow integration
- local testing infrastructure
- documentation discipline
- practical execution across backend and client surfaces

## Public Portfolio Guidance

When showing this repo publicly:

- present it as collaborative
- emphasize applied ML and system thinking
- be explicit that the project is personalized to one bird
- state clearly that the repo is a prototype and research-engineering artifact
- separate real implementation from still-unproven evaluation claims
